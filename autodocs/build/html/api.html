Site¶

Controls the details of the program execution.

class wallymart.site.pages.Pages¶

Controls the details of the program execution. Each method is implemented as a series of while-loops that asks for user input and returns any objects required by the main program.

The methods in this class hold pages that can be accessed by both the customer and employee. All methods are class methods, so they do not require instantiating the class to run the method. A reference to the logger in wallymart.wallymart_app.WallymartApp is saved as a class attribute so that the same logger instance is used throughout the program.

The Pages class is a superclass that inherits from wallymart.site.portal.customer_portal.CustomerPortal and wallymart.site.portal.employee_portal.EmployeePortal, which hold customer-specific or employee-specific pages, enabling wallymart.wallymart_app.WallymartApp to just import one object.

class wallymart.startup.database_configurator.DatabaseConfigurator(data_dir='data')¶

Responsible for creating any tables in the database that do not exist.

Variables

  • filename_for_tablename (dict) – This determines the filenames of the tables to be created.

  • data_dir (str) – This is folder containing the database tables. It is set to “data” by default, but can be changed from WallymartApp.

initialize_database()¶

class wallymart.startup.logger_configurator.LoggerConfigurator(logger=None, log_dir='log', filename='log')¶

Responsible for instantiating the logger used throughout the lifecycle of the main program. The logger is configured with a stream_handler, which displays messages to the console, and a file_handler, which outputs the messages to a log file.

Variables

  • msg_format (str) – Formats the output string

  • msg_format – Formats the date. Default is “%Y%m%d %H:%M:%S”

  • logger (Logger) – Stores an instance of the logger class.

  • log_dir (str) – The directory of log files.

  • filename (str) – The base filename of the log file. Note that the actual filename will include the start timestamp of the main program.

  • level – Can use this to toggle between logging.INFO and logging.DEBUG

add_file_handler()¶

Enables logger to write to the log file

add_stream_handler()¶

Enables logger to output to the console.

class wallymart.utils.credential_encoder.CredentialEncoder(item, hash_algorithm='md5', encoding='utf-8')¶

This is called by the Credentials class to hash an item prior to storing it. This is great for security. However, one-way hashing means the user will be unable to retrieve their passsword if they forget. In this scenario, the user should make a new account. Otherwise, a database administrator can help with account deletion.

Parameters

  • item (str) – Stores the item to be hashed. The item is hashed at initialization.

  • hash_algorithm (str) – Choose between “md5” and “sha256”

  • encoding (str) – Default is ‘utf-8’, but you can change this to arrive at a different hash.

class wallymart.orm.product.Product(product_name='', description='', quantity=0, price=0)¶

Container used to hold product information prior to writing it to the database. This information is entered by the employee and updated using the appropriate getters and setters.

Variables

  • product_name (str) – The name of the product. This must be unique, since it is used to identify the product.

  • description (str) – A short description of the product.

  • quantity (int) – How much inventory is in stock.

  • price (float) – This is rounded to the nearest cent.
